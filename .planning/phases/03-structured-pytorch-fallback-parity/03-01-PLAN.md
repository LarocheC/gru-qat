From `src/gru_qat/structure.py` (lines 207-225):

`class _CirculantLinear(nn.Module)`:
  - `__init__(self, n: int, *, bias: bool = False) -> None`: registers `self.n = n`, `self.col = nn.Parameter(torch.randn(n) / (n ** 0.5))`, and `self.bias = nn.Parameter(torch.zeros(n))` if `bias=True` else `register_parameter("bias", None)`.
  - `forward(self, x: torch.Tensor) -> torch.Tensor`:
      ```
      col_f = torch.fft.rfft(self.col)
      x_f   = torch.fft.rfft(x, dim=-1)
      y     = torch.fft.irfft(col_f * x_f, n=self.n, dim=-1)
      ```
      Implements `y[b, k] = sum_j col[(k - j) mod n] * x[b, j]` — circular convolution of `col` with `x`. The corresponding circulant matrix `C` has `C[i, j] = col[(i - j) mod n]` (first column = col); `y = x @ C.T` is the row-vector idiom.

Validator (lines 89-98): circulant requires `in_features == out_features` AND `n` is power-of-2 (`_is_pow2`).
