<objective>
Create `tests/test_triton_diagonal_strict.py` — the strict-tier (< 1e-5 abs under `torch.set_float32_matmul_precision('highest')`) parity audit for the diagonal Triton scan kernel. Diagonal has no matmul on the hidden side, so the kernel is naturally tight (the existing TF32-tier file already passes < 1e-4 in many cases per PATTERNS.md). Strict tier should pass cleanly for the same reason — diagonal recurrence drift comes only from per-step nonlinearities, which are bit-stable in fp32 IEEE.

Per D-16, this is the ONLY new strict-tier file that includes H ∈ {1, 2} cases — those are the diagonal-kernel-specific edge cases. Other kernels' tiny-H exploration is Phase 6.

Purpose: TRI-02 (diagonal Triton fwd+bwd at < 1e-5) needs strict-tier coverage that doesn't exist in `tests/test_triton_diagonal.py` (which is TF32/realistic-tier).

Output: One new test file with 4 distinct test functions (fwd + bwd × fast + slow). Skips cleanly on CPU; ready for Plan 02-06 GPU run.
</objective>
