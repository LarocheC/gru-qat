    Append two new parametrized test functions to `tests/test_layer_parity.py`, AFTER the h_T tests from Plan 02. No new imports required.

    Function 1 — `test_layer_backward_matches_nn_gru(T: int, B: int, H: int) -> None`. Decorator: `@pytest.mark.parametrize("T,B,H", FAST_GRID)`. Body structure:

    Step 1 (setup, identical to forward test): `torch.manual_seed(0)`, `IN = max(H, 1)`, `layer = _make_dense_fp32_layer(IN, H)`, `gru = _translate_cell_to_nn_gru(layer)`.

    Step 2 (build two separate `requires_grad=True` leaf tensors from the same seed-derived data so each implementation owns its own autograd graph): `x_ref = torch.randn(T, B, IN, requires_grad=True)`. `h0_ref = torch.zeros(1, B, H, requires_grad=True)` (nn.GRU shape, with leading num_layers axis). `x_ours = x_ref.detach().clone().requires_grad_(True)`. `h0_ours = h0_ref.detach().squeeze(0).clone().requires_grad_(True)` (GRULayer shape, with the leading axis stripped). NOTE: must squeeze BEFORE clone so the clone's storage matches the [B, H] shape and `.grad` accumulates with that shape.

    Step 3 (forward both paths): `out_ref, _ = gru(x_ref, h0_ref)`. `out_ours, _ = layer(x_ours, h0_ours)`. Discard h_T returns — only `out` is used to drive backward (`out.backward(g)` with g shaped like out is the standard pattern, and h_T is fully determined by the last `out` row so the gradient signal does cover it).

    Step 4 (shared downstream gradient): `g = torch.randn_like(out_ref)`. Then `out_ref.backward(g)` and `out_ours.backward(g)` — same g sent into both autograd graphs.

    Step 5 (per-param assertions). Build a list of `(name, ref_tensor, our_tensor)` triples and loop:
    - `("dx", x_ref.grad, x_ours.grad)`
    - `("dh_0", h0_ref.grad.squeeze(0), h0_ours.grad)` — note the squeeze on the ref side.
    - For dW_ih, dW_hh, db_ih, db_hh: build the cell-side cat tensors INSIDE the test, e.g. `cell = layer.cell; our_W_ih = torch.cat([cell.W_ir.grad, cell.W_iz.grad, cell.W_in.grad], dim=0)` etc. Then add `("dW_ih", gru.weight_ih_l0.grad, our_W_ih)`, same for `weight_hh_l0.grad` ↔ cat of three W_h*.grad, and the two bias families.
    - Loop body: `rel = (ref_t - our_t).abs().max().item() / max(ref_t.abs().max().item(), 1e-6); assert rel < 1e-4, f"{name} rel diff {rel:.4e} (T={T},B={B},H={H})"`. The `{name}` token in the failure message is the single most diagnostic thing in this whole plan — it points at exactly which of the six gradient tensors drifted.

    Docstring: "Backward parity: all six weight grads (concatenated against nn.GRU's [3H, IN] / [3H, H] layouts), both bias grads, dx, and dh_0 match nn.GRU autograd to < 1e-4 across the fast grid. Separate from forward parity per D-09: if forward passes but backward fails, the bug is in the autograd graph for the backward step (e.g. the n-gate's `r * gh_n` derivative), not the forward math."

    Function 2 — `test_layer_backward_matches_nn_gru_slow(T: int, B: int, H: int) -> None`. Identical body. Decorators: `@pytest.mark.slow` ABOVE `@pytest.mark.parametrize("T,B,H", SLOW_GRID)`. Same docstring but note "slow grid".

    Implementation notes:
    - Do NOT reuse `x_ref` as the input to GRULayer — autograd would build one tape across both, and `.backward(g)` would crash on a second-backward. The detach-clone pattern is mandatory.
    - Do NOT call `.zero_grad()` anywhere — both implementations start with `.grad = None` and the first backward populates them. If `.zero_grad()` were called on the layer, we'd lose the cell's grads. (The same holds for the gru.)
    - Do NOT accept `loss = out.sum()` as a shortcut — using `out.backward(g)` with a random g is more discriminating because every output element contributes independently to the gradient signal.
    - The `1e-6` denominator floor matches all other relative-error idioms in the file. For grids where `H=1, B=1, T=1` the ref-grad max may be small enough that the floor matters.

    Do NOT add `xfail`. Do NOT touch the helpers from Plan 01 or the grid constants from Plan 02. Do NOT fuse with the forward/h_T tests.
